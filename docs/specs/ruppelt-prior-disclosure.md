# Ruppelt 公開情報ステータス仕様

Ruppelt の公開情報ステータスは、PURSUE / war.gov の公式メタデータを正本とし、外部アーカイブや手動照合結果を補助情報として扱う。

## データ方針

- war.gov CSV 由来の値は `source` に保持する。
- 日本語訳は `ja` に保持する。
- 公開状況の判定は `priorDisclosure` に保持し、`source` には混ぜない。
- 検索用の派生値は `searchFacets` に保持する。
- 「元データ不備」は独立ステータスにせず、MVPでは `unknown` / `判定不能` に含める。
- `priorDisclosure` が未投入のレコードは、UIでは `未照合` と表示する。これは判定ステータスではなく、まだ手動判定データがないことを示す表示状態。

## ステータス

- `first_time_public`: 初公開
- `previously_public`: 既に公開済み
- `partial`: 一部公開済み
- `unknown`: 判定不能

## 判定ルール

### 初公開

主要な公的アーカイブ・既存公開資料で一致が確認できない場合。ただし、必ず信頼度を持たせる。

### 既に公開済み

同じ文書、画像、音声、映像、または同等の全文資料が過去に公開されていた場合。

### 一部公開済み

元資料・元画像・元音声・事件の一部が既に公開済みだが、PURSUE版に新しい注釈、編集、文脈付け、または追加ファイルがある場合。

### 判定不能

OCR不能、照合材料不足、タイトルやメタデータが曖昧、外部アーカイブで確認できない場合。PURSUE公式データのファイル不整合、リンク切れ、重複、誤ったメタデータ、ミラー不備もMVPではここに含める。

## 表示方針

- カードでは小さく `公開状況：判定不能 ⓘ` のように表示する。
- `priorDisclosure` 未投入のカードでは `公開状況：未照合 ⓘ` と表示し、公開状況フィルターの `判定不能` には含めない。
- 詳細パネルでは、公開状況、信頼度、判定材料、確認リンク、補足メモ、参考照合元を表示する。
- `Ruppelt確認済み` は `ruppeltVerified: true` の場合だけ表示し、未確認表示は出さない。
- UI上では「アトリビューション」という言葉を使わない。
- they-are-here.com を使う場合は、下部に小さく `参考照合元：they-are-here.com` と表示する。
- they-are-here.com や外部サイトの文章を大量転載しない。

## Release 02 暫定照合

- Release 02 は Ruppelt 側の暫定照合として扱う。
- 近年のDOW動画は、war.gov / DVIDS公式公開以外の同一動画公開が確認できない場合、低信頼度の `初公開` とする。
- ただし、日付・場所などの照合メタデータが不足しているDOW動画、または公式URLが複数レコードで重複しているDOW動画は、弱い照合として `判定不能` に寄せる。
- NASA音声は、ミッション記録・トランスクリプトなど関連記録が既公開の場合、PURSUE版音声抜粋そのものの過去公開が未確認でも `一部公開済み` とする。
- PANTEX資料のようにPURSUE以前の同一または同等資料が確認できる場合は `既に公開済み` とする。
- CIA/DOE等の文書で同一資料の過去公開が確認できない場合は、無理に初公開とせず `判定不能` にできる。
